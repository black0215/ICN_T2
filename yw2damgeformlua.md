## 2) 공통 난수 래퍼 공식 (확정)



## 2-1. `0x0045E748` : `roll_0_to_N`



관찰된 로직:



```c

uint32_t roll_0_to_N(uint32_t rngStateId, uint32_t N) {

    if (N == 0) return 0;

    // RNG core: 0x51EC28

    float x = rng_float_scaled(rngStateId, (float)(N + 1));

    uint32_t r = (uint32_t)x;

    if (r > N) r = N;

    return r; // 0..N (inclusive)

}

```



따라서 사건 확률은 다음과 같다.



- `P(roll == k) ~= 1/(N+1)` (`0 <= k <= N`)

- `P(roll <= T) = (T+1)/(N+1)`

- `P(roll < T) = T/(N+1)`

- `P(roll >= T) = (N-T+1)/(N+1)`



`N=99`일 때 1스텝은 1% 단위이다.



## 2-2. `0x0045DC1C` : `roll_min_to_max`



관찰된 로직:



```c

uint32_t roll_min_to_max(uint32_t rngStateId, uint32_t minV, uint32_t maxV) {

    if (minV >= maxV) return 0;

    uint32_t range = maxV - minV;

    uint32_t r = roll_0_to_N(rngStateId, range); // 0..range

    return r + minV; // minV..maxV (inclusive)

}

```

## 3) 데미지 공식 (최종 클램프) 정리



## 3-1. 최종 상한 999 클램프 A (확정)



핵심 코드:



- `0x00470250: cmp r4, #0x3E8` (1000 비교)

- `0x00470254: ldrge r4, [pc, #0x24]`

- `0x00470280: 0x000003E7` (999)



즉:



```c

if (r4 >= 1000) r4 = 999;

```



또한 동일 경로에서 조건부 최소값 보정 존재:



- `0x00470244: cmp r4, #1`

- `0x00470248: movlt r4, #1` (일부 분기에서만)



정리:



```text

D_final_pathA = clamp(D_raw, lowerA, 999)

lowerA in {0,1}  // 분기 의존

```



## 3-2. 최종 상한 999 클램프 B (확정)



핵심 코드:



- `0x00471028: cmp r4, #0x3E8`

- `0x0047102C: bhs 0x00471018`

- `0x00471018: mov r4, sl` (`sl`은 `0x00470EAC`에서 999 로드)

- `0x00471020~0x00471034`: `r4==0`이면 `r4=1`



정리:



```text

D_final_pathB = clamp(D_raw, 1, 999)

```



## 3-3. 데미지 공식 골격 (강한 추정)



정적 분석상 다수 버프/상태 분기 후 최종 `r4`를 산출하고, 마지막에 위 두 경로 중 하나로 클램프한다.



```text

D_raw = F(basePower, statDelta, skill/ability modifiers, RNG terms, context flags)

D_final = clamp_by_path(D_raw)

```



여기서 상한은 전투 경로 기준으로 실질적으로 `999`로 고정된다.

## 4) 명중/회피/치명타 확률식



## 4-1. 0..99 롤 기반 판정 (확정)



여러 전투 함수에서 공통으로:



```asm

mov r1, #0x63

mov r0, #0

bl  0x45E748   ; roll in [0,99]

```



즉 100분할 확률 단위 사용이 확인된다.



## 4-2. 명중/회피 판정 후보식 A (`0x00472EDC` 주변, 강한 추정)



관찰된 핵심 분기:



```asm

cmp  r4, #1

movlt r4, #1

...

roll = rand(0..99)

cmp  roll, r4

blt  fail

success

```



식:



```text

P_success = P(roll >= r4)

          = (100 - r4) / 100    (1 <= r4 <= 99)

```



경계:



- `r4 <= 1`이면 약 `99%`

- `r4 >= 100`이면 `0%`



## 4-3. 명중/회피/보정 판정 후보식 B (`0x0046F05C` 주변, 강한 추정)



관찰된 핵심:



```asm

roll = rand(0..99)

L = (100 - r5) * r6

if (L > 100*roll) fail else success

```



동치식:



```text

success <=> roll >= ceil(L/100)

P_success = clamp((100 - ceil(L/100))/100, 0, 1)

```



`r5`, `r6`의 정확한 의미(회피/명중/상태 보정 파라미터)는 추가 추적 필요.



## 4-4. 치명타 판정 후보식 (`0x00471568` 주변, 강한 추정)



관찰된 핵심:



```asm

N = 99 * r5

roll = rand(0..N)

T = r5*(r4 + 1)

if (roll < T) success else fail

```



식:



```text

P_success = T / (N+1)

          = r5*(r4+1) / (99*r5 + 1)

```



특수 케이스:



- `r5=1`이면 `P_success = (r4+1)/100`

## 8) 결론 요약



1. 전투 최종 데미지 상한은 코드상 `999` 클램프가 직접 존재한다.

2. 확률 판정은 `rand(0..99)` 기반 1% 단위 구조가 공통적으로 확인된다.

3. 명중/회피/치명타는 각각 전용 후보 루틴이 분리되어 있으며, 효과 해시(`0x019BED9D`, `0x9892BC27`, `0xEF958CB1`, `0x512CB8A3`)와 연결된다.

4. "공식"은 현재 단계에서 `최종 클램프식`은 확정, `세부 내부항(파라미터 의미)`은 부분 미확정이다.

## 9) 수식 자연어 요약



1. 공통 난수 공식

- `roll_0_to_N`은 0부터 N까지를 포함해서 하나를 뽑는다.

- 그래서 N이 99면 0부터 99까지, 총 100칸에서 하나를 뽑는 구조다.

- `roll_min_to_max`는 min부터 max까지를 포함해서 하나를 뽑는다. 내부적으로는 `0..(max-min)`을 뽑은 뒤 min을 더한다.



2. 데미지 최종값 공식

- 전투 내부 계산으로 먼저 원시 데미지(`D_raw`)를 만든다.

- 마지막 단계에서 경로에 따라 최소값을 0 또는 1로 보정한 뒤, 최대값은 999로 잘라낸다.

- 즉 실제 최종 데미지는 경로 A에서는 `0~999`, 경로 B에서는 `1~999` 범위를 넘지 않는다.



3. 명중/회피 확률 공식(요약)

- 명중/회피 계열 판정은 기본적으로 0부터 99까지 난수를 뽑아 임계값과 비교하는 방식이다.

- 임계값이 높아질수록 성공 확률은 내려가고, 임계값이 낮아질수록 성공 확률은 올라간다.

- 0..99를 쓰기 때문에 확률 해상도는 1% 단위다.



4. 치명타 확률 공식(요약)

- 치명타 후보 루틴은 먼저 범위형 난수를 뽑고, 별도로 계산된 임계값보다 작은지를 비교한다.

- 임계값이 커질수록 치명타 성공 확률이 올라간다.

- 특정 단순 케이스(`r5=1`)에서는 사실상 `(r4+1)%` 형태로 해석된다.



5. 효과(어빌리티)와 확률의 연결

- 명중 강화는 `0x019BED9D`, 회피 강화는 `0x9892BC27`, 치명 강화는 `0xEF958CB1`, 양면형(치명/피격 리스크)은 `0x512CB8A3` 해시로 연결된다.

- 다만 텍스트 단계(조금/보통/매우)와 실제 내부 계수 매핑은 일부가 동일 템플릿을 공유하므로, 최종 강도는 상위 분기 맥락까지 같이 봐야 한다.

## 10) 수학식 버전 (Natural Language -> Math)



기호 정의:

- `R ~ U{a, ..., b}`: 정수 균등분포

- `1[cond]`: 조건이 참이면 1, 거짓이면 0

- `clamp(x,l,u) = min(max(x,l),u)`

- `floor(x)`: 내림



### 10-1. 공통 난수 래퍼



1. `roll_0_to_N`



- 입력: `N ∈ Z_{>=0}`

- 난수: `u ∈ [0,1)`

- 출력:



`r = min(floor((N+1)u), N)`



따라서



`r ~ U{0,1,...,N}`



`P(r = k) = 1/(N+1)   (0 <= k <= N)`



`P(r <= T) = (T+1)/(N+1)`



2. `roll_min_to_max`



- 입력: `m, M ∈ Z`

- 출력:



`roll(m,M) = 0                        (m >= M)`



`roll(m,M) = m + r,  r ~ U{0,...,M-m} (m < M)`



즉



`roll(m,M) ~ U{m,...,M}`



### 10-2. 최종 데미지 상한식



1. 경로 A



`D_final_A = clamp(D_raw, L_A, 999),  L_A ∈ {0,1}`



2. 경로 B



`D_final_B = clamp(D_raw, 1, 999)`



### 10-2b. 엔진 실코드 기반 최종 데미지 계산식 (`0x0046F158`, `0x00470DFC`)



아래는 `0x0046F158` 본체에서 실제로 확인된 상수/연산을 기준으로 정리한 식이다.



- 확인된 상수:

  - `1.0` (`0x3F800000`)

  - `0.0` (`0x00000000`)

  - `0.01` (`0x3C23D70A`)

  - `0.2` (`0x3E4CCCCD`)

  - `0.5` (`0x3F000000`)

  - `0.25` (`0x3E800000`)

  - `0.1` (`0x3DCCCCCD`)

  - `2/3` (`0x3F2AAAAB`)

  - `0.4` (`0x3ECCCCCD`)

  - `1/14` (`0x3D924925`)

  - `1/3` (`0x3EAAAAAB`)

  - `1/7` (`0x3E124925`)



기호:

- `A`: 공격측 기본량(`sub_563C20` 또는 분기 시 `sub_563BF8` 계열 출력)

- `P`: 기준 파워(호출자에서 전달된 정수, float 변환 사용)

- `L`: `((flags_1E >> 11) & mask)` 계열 레벨/계수 항

- `M`: `sub_55FE70(...)` 계열 배율

- `B`: 추가 보정 누적(`s21`)

- `S`: 추가 가산율 누적(`s20`)

- `k = 0.01`



1. 공통 코어항(주 경로)



`X = A + M*L`



`D0 = (2/3)*X - 0.1*P` (분기: `sp+0x8C != 0`)



`D0 = (1/2)*X - 0.25*P` (분기: `sp+0x8C == 0`)



2. 대체 코어항(특수 타입 분기: `0x0046F9D4` 이후)



`X = A + M*L`



`D0 = ((0.4)*X - (1/14)*P + (0.2)*X) * E` (분기: `sp+0x8C != 0`)



`D0 = ((1/3)*X - (1/7)*P + (0.1)*X) * E` (분기: `sp+0x8C == 0`)



`E`는 타입/원소 테이블 분기(`sub_560A50`)에서 얻는 계수.



3. 공통 보정 결합



`D1 = D0 + (0.2*X)*(B - 1)`



`D2 = D1 * Π_i (1 + k*c_i^+) * Π_j clamp(1 - k*c_j^-, 0, 1)`



필요 조건에서:



`D2 = D2 / L`



4. 후반 스프레드/카테고리 보정



`D3 = D2 * (1 + S)`



카테고리 분기(`sp+0x88`):



`D4 = ceil(D3 * μ)` (`type in {1,3}`, 기본 `μ=0.5`, 특정 분기 `μ=0.25` 또는 해시 기반 계수)



`D4 = ceil(D3 * max(0, 100 - T)/100)` (`type == 2`, `T`는 테이블 값)



`D4 = D3` (그 외)



5. 정수화 및 최종 반환 전 처리



`Dint = int(D4)` (일부 분기)



또는



`Dint = int(0.9*D4 - 1)` (특정 플래그 분기)



또는



`Dint = int(F_rand(D4))` (`sub_46DF50` 경로)



이후:



`Dint = clamp(Dint, 0, HP_cap)` (특정 조건)



`Dint = max(Dint, 1)` (특정 조건)



`Dret = min(Dint, 999)` (`0x00470250`)



즉 일반 경로 요약식은:



`Dret = min(999, PostClamp(Intize(CategoryScale((Core(A,P,L,M)*BuffMul + AddTerm) * (1+S)))))`



6. `0x00470DFC` 상위 분기에서의 특수 최종식



`0x0046F158` 호출 이전/외부 분기에서는 아래 식이 직접 사용되기도 한다.



- `D = u16(rand0 * Sum * 0.01)` (`0x00470F10`~`0x00470F30`)

- `D = u16(scale * Value * 0.01)` (`0x00470FC8`~`0x00470FE4`)

- `D = max(0, Base - floor(F))` (`0x0047104C`~`0x0047106C`)



이 특수 분기들도 이후 동일하게 `1`/`999` 경계 보정을 받는 경로가 존재한다.



### 10-3. 명중/회피 판정식 후보 A (`0x00472EDC` 계열)



- 난수: `R ~ U{0,...,99}`

- 임계값 보정:



`H = clamp(H_raw, 1, 100)`



- 성공 조건:



`Hit = 1[R >= H]`



- 성공 확률:



`P(Hit=1) = max(0, (100 - H)/100)`



### 10-4. 명중/회피 판정식 후보 B (`0x0046F05C` 계열)



- 난수: `R ~ U{0,...,99}`

- 내부 값:



`L = (100 - r5) * r6`



- 성공 조건:



`Hit = 1[L <= 100R] = 1[R >= ceil(L/100)]`



- 성공 확률:



`c = ceil(L/100)`



`P(Hit=1) =`



`1                      (c <= 0)`



`(100 - c)/100          (1 <= c <= 99)`



`0                      (c >= 100)`



### 10-5. 치명타 판정식 후보 (`0x00471568` 계열)



- 난수 범위:



`N = 99 * r5`



`R ~ U{0,...,N}`



- 임계값:



`T = r5 * (r4 + 1)`



- 성공 조건:



`Crit = 1[R < T]`



- 성공 확률:



`P(Crit=1) = clamp(T/(N+1), 0, 1)`



특수 케이스 `r5=1`:



`P(Crit=1) = clamp((r4+1)/100, 0, 1)`



### 10-6. 효과 해시를 포함한 추상식



정밀 파라미터 매핑이 미완료인 구간은 추상식으로 표현하면 다음과 같다.



1. 명중 임계값:



`H_raw = H_base + Σ_i alpha_i * E_i`



`H = clamp(H_raw, 1, 100)`



`P_hit = max(0, (100 - H)/100)`



2. 치명 임계값:



`T_raw = T_base + Σ_j beta_j * C_j`



`P_crit = clamp(T_raw/(N+1), 0, 1)`



여기서 `E_i, C_j`는 효과 해시(`0x019BED9D`, `0x9892BC27`, `0xEF958CB1`, `0x512CB8A3`)와 문맥(상태/버프/스킬)에서 유도되는 항이다.
