I will implement the **Character Scale UI** (`CharacterScaleView`) by referencing the style of `CharacterInfoV3.xaml` and the logic from the legacy `CharascaleWindow`.

### 1. ViewModel Implementation (`CharacterScaleViewModel.cs`)
*   **Purpose**: Manage data logic for Character Scales.
*   **Data Loading**:
    *   Load `CharScale[]` via `IGame.GetCharascale()`.
    *   Load `CharaBase[]` via `IGame.GetCharacterbase()` to map `BaseHash` to Character Names and Icons.
    *   Create a `CharacterScaleWrapper` class to combine `CharScale` data with `CharaBase` visual info (Name, Rank, Tribe, Icon).
*   **Features**:
    *   **Search/Filter**: Filter by Name, Hash, Tribe, Rank (similar to CharacterInfoV3).
    *   **Editing**: properties for `Scale1` (Map), `Scale2` (Companion), `Scale3` (Ally Battle), `Scale4` (Enemy Battle), `Scale5` (Menu), `Scale6` (Busters), `Scale7` (Extra).
    *   **Saving**: `SaveChanges()` method to call `IGame.SaveCharascale()`.

### 2. View Implementation (`CharacterScaleView.xaml`)
*   **Style**:
    *   Clone the 3-column layout of `CharacterInfoV3.xaml`.
    *   Use the same **PuniPuniTheme** resources (Glassmorphism effects, semi-transparent backgrounds).
*   **Left Panel**:
    *   Search Box and Filter options.
    *   List of characters (displaying Icon, Name, Rank/Tribe) derived from the mapped `CharaBase`.
*   **Right Panel (Detail Area)**:
    *   **Header**: Selected Character Icon, Name, and Hash.
    *   **Controls**: 7 sets of Sliders and Numeric Input fields for adjusting the scales.
    *   **Labels**: "World Map", "Companion", "Battle (Ally)", "Battle (Enemy)", "Menu", "Busters", "Extra".
