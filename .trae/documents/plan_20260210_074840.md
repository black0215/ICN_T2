I will implement the **Yokai Stats Editor UI** (`YokaiStatsView`) combining **iPhone (rounded cards, smooth navigation)** and **Windows 11 (Mica-like transparency, clean typography)** design elements.

### 1. ViewModel Implementation (`YokaiStatsViewModel.cs`)

* **Purpose**: Manage `YokaiStats` data logic.

* **Data Loading**:

  * Load `YokaiStats[]` via `IGame.GetCharaparam()`.

  * Load `CharaBase[]` via `IGame.GetCharacterbase()` to map `BaseHash` to Names and Icons.

  * Create `YokaiStatsWrapper` to combine stats data with visual info (Name, Icon, Rank, Tribe).

* **Features**:

  * **Search/Filter**: Filter by Name, Hash, Tribe, Rank.

  * **Grouping**: Organize properties into logical categories (Basic, Stats, Battle, Drops, Evolution).

  * **Saving**: `SaveChanges()` to call `IGame.SaveCharaparam()`.

### 2. View Implementation (`YokaiStatsView.xaml`)

* **Style**:

  * **Background**: Deep acrylic/glass effect (`#50FFFFFF`) over the main window background.

  * **Layout**: 3-column layout (Search List | Spacing | Detail Area).

* **Left Panel (Navigation)**:

  * **Search**: Rounded search box with clear button.

  * **List**: Virtualized list with "iPhone Settings" style rows (Icon + Title + Subtitle + Chevron).

* **Right Panel (Detail)**:

  * **Header**: Large Icon, Name (Bold), Hash (Monospace).

  * **Tabs**: Segmented Control (Pill shape) for "Basic", "Stats", "Battle", "Drops".

  * **Content Area (Cards)**:

    * **Stats Card**: 2-column grid for HP, Str, Spr, Def, Spd (Min/Max inputs).

    * **Attributes Card**: Visual sliders/bars for elemental resistance.

    * **Skills Card**: Dropdowns for Attack, Technique, Soultimate.

    * **Drops Card**: List of drop items with probability inputs.
