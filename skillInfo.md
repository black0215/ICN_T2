## 1) 문서 목적 / 독자 / 범위

1. 목적
- `yw2_effect_info_grouped_by20_v1.json`의 항목별 의미와 값 해석 기준을 실무용으로 정리한다.

2. 주 독자
- 모드 제작자(효과 해시 교체, 강도 조정, 안전 편집이 필요한 사용자)

3. 범위
- JSON 포맷 설명 + 필드별 의미 + 값별 해석 + 편집 규칙
- 코드/바이너리/JSON 원본 수정은 포함하지 않는다.

4. 확신도 표기 규칙
- `[확정]`: 구조/매핑/빈도가 직접 확인된 사실
- `[강한 추정]`: 통계/비교 근거가 강한 해석
- `[가설]`: 실험/추가 리버싱이 필요한 해석

## 2) JSON 전체 구조 맵

```text
root
├─ meta
│  ├─ generatedAt
│  ├─ baseAsset
│  ├─ textAsset
│  ├─ totalSkills
│  ├─ groupUnit
│  └─ notes
├─ fieldGuessReference[]         // v1~v17 해석 가설/근거
└─ groups[]
   ├─ range                      // 예: "1-20"
   ├─ skillCount
   ├─ effectEntryCount
   ├─ fieldValueTopCounts        // 그룹 단위 v1~v17 상위 빈도
   └─ items[]
      ├─ skillIndex
      ├─ configRow0Based
      ├─ configHash
      ├─ nameHash
      ├─ descriptionHash
      ├─ col3
      ├─ name
      ├─ description
      ├─ usedInCharaParam
      └─ effects[]
         ├─ effectSlot
         ├─ effectDataRow
         ├─ effectHash
         ├─ effDataP1toP8[]
         ├─ effDataNonZeroIndices[]
         ├─ effectInfoRow
         ├─ effectInfoVectorCompact[]
         ├─ effectInfoVector{v1..v17}
         └─ sharedByAbilities[]
```

## 3) Top-level 키 해설

| 키 | 타입 | 의미 | 비고 |
|---|---|---|---|
| `meta` | object | 파일 생성 메타 | `[확정]` 생성 스크립트에서 직접 작성 |
| `fieldGuessReference` | array | `v1~v17` 의미 가설 사전 | `[확정]` 보고용 메타 데이터 |
| `groups` | array | 20개 단위 묶음 데이터 | `[확정]` `groupUnit=20` 고정 |

## 4) `groups[*]` 해설

| 키 | 타입 | 의미 | 실무 포인트 |
|---|---|---|---|
| `range` | string | 스킬 인덱스 구간 | 예: `205-220` |
| `skillCount` | number | 구간 내 스킬 개수 | 마지막 그룹만 18 |
| `effectEntryCount` | number | 효과 엔트리 수 | 스킬당 1개가 대부분 |
| `fieldValueTopCounts` | object | 그룹별 `v1~v17` 상위 빈도 | 템플릿 구간 파악에 유용 |
| `items` | array | 실제 스킬 레코드 | 편집 대상 본문 |

[확정] 전체 규모: 스킬 `358`, 효과 엔트리 `359`, 그룹 `18`.

## 5) `items[*]` 해설

| 키 | 타입 | 의미 | 실무 포인트 |
|---|---|---|---|
| `skillIndex` | number | 1-based 스킬 인덱스 | 원문 note와 일치 |
| `configRow0Based` | number | 0-based config row | 바이너리 row 대응 |
| `configHash` | hex string | 능력 식별 해시 | `chara_param` 참조 키 |
| `nameHash` | hex string | 이름 해시 | 텍스트 테이블 매핑용 |
| `descriptionHash` | hex string | 설명 해시 | 텍스트 테이블 매핑용 |
| `col3` | hex string | 원본 col3 | 의미 미확정, 유지 권장 |
| `name` | string | 해석된 이름 | 없으면 hex fallback |
| `description` | string | 해석된 설명 | 일부 공백 가능 |
| `usedInCharaParam` | bool | 실제 캐릭터 데이터 사용 여부 | true 우선 편집 권장 |
| `effects` | array | 해당 스킬의 효과 목록 | 대부분 1개 |

[확정] 사용 분포: `usedInCharaParam true=190 / false=168`.

## 6) `effects[*]` 해설

| 키 | 타입 | 의미 | 실무 포인트 |
|---|---|---|---|
| `effectSlot` | number | 스킬 내 효과 슬롯 인덱스 | 0부터 시작 |
| `effectDataRow` | number | `EFF_DATA` row | 직접 매핑 추적용 |
| `effectHash` | hex string | 효과 동작 식별자 | 가장 중요한 키 |
| `effDataP1toP8` | string[8] | 개별 오버라이드 파라미터 | 대부분 0 |
| `effDataNonZeroIndices` | number[] | `EFF_DATA` 비영(非0) 필드 인덱스 | diff 확인용 |
| `effectInfoRow` | number\|null | `EFFECT_INFO` row | null이면 미해결 |
| `effectInfoVectorCompact` | string[17]\|null | `v1~v17` compact 배열 | 분석/클러스터링용 |
| `effectInfoVector` | object\|null | `v1..v17` named object | 사람이 읽기 쉬움 |
| `sharedByAbilities` | string[] | 같은 `effectHash` 공유 스킬명 | 파급 영향 확인 |

[확정] `effectHash`는 `EFF_DATA[0]`과 `EFFECT_INFO[0]`를 잇는 핵심 키다.

### 9-3. `v1/v2` 조합 해석

- `[강한 추정]` `v1`은 효과 카테고리 축으로 작동한다.
- `[가설]` `v2`는 대상/스코프 분기 축일 가능성이 높다.
- `[가설]` `v1=2, v2=0` 조합은 전투형 템플릿에서 자주 보인다.

### 9-4. `P8` 값 해석

- `[확정]` `P8=0`은 일반 구간의 기본값이다.
- `[강한 추정]` `P8=10`은 후반 확장/테스트 템플릿 집합 마커다.

## 11) 모드 제작자용 실전 사용 규칙

1. 우선순위
- `usedInCharaParam=true` 항목만 먼저 편집한다.

2. 안전한 수정 순서
- 1단계: `effectHash` 교체(동작 축)
- 2단계: `v8` 소폭 조정(강도 축)
- 3단계: 필요 시 `P3`, `P4`, `P5` 최소 조정

3. 권장 금지 항목
- `v13~v17`, `P6`, `P7`는 근거가 약하므로 초기 실험에서 건드리지 않는다.

4. 멀티효과 편집 규칙
- `effectSlot` 순서를 바꾸지 않는다.
- 슬롯별 해시 의미를 분리 검증한 뒤 결합한다.

5. 후반 구간 작업 규칙
- `P8=10` 구간(205~331)은 템플릿성 데이터가 많으므로 실전 적용 전에 별도 검증한다.
